01 / The Basics

What is an LLM?

LLM stands for Large Language Model. It's the technology behind ChatGPT, Claude, Gemini, and most AI assistants you've heard of.

Think of it this way: Imagine reading every book, article, Wikipedia page, and website ever written — billions of pages of text. After reading all of that, you'd get very good at predicting what comes next in a sentence, understanding questions, and generating helpful answers. That's essentially what a language model does, except a computer does the reading.

The model doesn't "look up" answers like a search engine. It generates text based on patterns it learned from training — word by word, predicting what comes next.

What are "tokens"? Hover over them:

Tokens are small pieces of text — roughly ¾ of a word on average. The model reads and generates tokens one at a time.

📚

Trained on text

LLMs learn from vast amounts of internet text, books, and code. They don't access the internet in real-time (unless that feature is added separately).

🔮

Predicts, doesn't retrieve

Unlike Google, an LLM doesn't search for an answer. It generates one based on learned patterns — which is why it can be wrong sometimes.

💡

Understands context

It remembers everything in the current conversation and uses that to give relevant, coherent responses.

02 / Memory

The Context Window

The context window is the AI's working memory — everything it can "see" at once in a conversation.

Analogy: Imagine having a conversation with someone who has a sticky note in front of them. They can only refer to what's written on that note. The context window is the size of that sticky note — everything in your current conversation fits on it. When the note is full, the AI starts to "forget" the oldest messages.
Context Window
You
Hi! Can you help me write a birthday message for my friend Sarah?
AI
Of course! Tell me a bit about Sarah — what does she love?
You
She loves hiking and her dog, Biscuit.
AI
Happy Birthday Sarah! May your trails be sunny and Biscuit's tail wag all day long 🐾
You
Now write one in Spanish please
AI
¡Feliz cumpleaños, Sarah! (The AI remembers Sarah's name and details from earlier in the window!)
Context used ~8% / 200,000 tokens
📏

Size matters

Context windows are measured in tokens. A 200K token window fits roughly 150,000 words — about 3 average novels!

Claude: 200K tokens GPT-4: 128K tokens Gemini: up to 1M tokens
🔄

Fresh start each time

By default, each new conversation starts with a completely empty context window. The AI doesn't remember last week's chat unless you paste it in.

📄

You can paste big docs

Because the window is large, you can paste an entire PDF, contract, or codebase directly into the chat and ask questions about it.

05 / Power Features

Customizing Claude Code

Claude Code isn't just an AI assistant — it's a platform you can shape to work exactly how you want. Here are four key ways to customize it.

📋 Rules — Teaching Claude your preferences

Rules are instructions you write once and Claude follows in every session. Store them in a file called CLAUDE.md in your project folder — Claude reads it automatically every time you start a conversation about that project.
Analogy: Think of it like a briefing document you hand to a new employee on their first day. Instead of repeating "we always use 2-space indentation" or "never delete files without asking" in every conversation, you write it once and Claude always knows.
# CLAUDE.md — My project rules

## Coding style
- Always use TypeScript, not JavaScript
- Prefer functional components in React
- Use 2-space indentation

## Safety rules
- Never delete files without asking me first
- Always run tests before committing

## Communication
- Keep explanations short — I'm an experienced developer
- When stuck, suggest 2-3 options instead of just one
🎨
Style rules "Always use single quotes", "Never use var, only const/let", "Tailwind CSS only"
🔒
Safety guardrails "Ask before running any shell command", "Never commit directly to main"
🧠
Context about your project "This is a healthcare app — be extra careful with data", "Users are elderly, use large fonts"

⚡ Skills — Creating custom slash commands

Skills are custom shortcuts you define that Claude can run on demand. Type /your-command in Claude Code and it executes a pre-written prompt or workflow — saving you from typing the same instructions repeatedly.
Analogy: Like keyboard shortcuts on your computer. Instead of going File → Save As → Browse → Select folder every time, you press Cmd+S. Skills let you say "/deploy" or "/review" instead of writing a long instruction each time.
# .claude/commands/review-pr.md

Review this pull request carefully:

1. Check for security vulnerabilities
2. Verify tests cover the new code
3. Look for performance issues
4. Check that error handling is thorough
5. Suggest improvements in a numbered list

# Now you can just type: /review-pr
🚀
/deploy-checklist Runs through your standard pre-deployment checks automatically
📝
/write-tests Generates comprehensive tests for whatever code you're looking at
📊
/standup Summarizes today's git commits into a standup-ready update

🪝 Hooks — Automating reactions to events

Hooks are shell commands that run automatically when specific events happen in Claude Code — like before a file is edited, after a command runs, or when Claude finishes a task. They let you automate quality checks, notifications, and workflows.
Analogy: Like the motion sensors in a smart home. When the front door opens (event), the lights turn on and you get a notification (hook). You set it up once, and it runs automatically every time.
{
  "hooks": {
    // Run linter automatically before any file edit
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{"type": "command", "command": "npm run lint"}]
      }
    ]
  }
}
🔍
Auto-lint before edits Run code style checks automatically every time Claude edits a file
🔔
Slack notification on completion Ping your team in Slack when Claude finishes a long task
🛡️
Security scanner Scan for secrets/passwords before any code is committed

🤖 Agents — Specialized AI assistants

Agents are specialized versions of Claude, each with a specific role and set of tools. Instead of one Claude doing everything, you can spin up a "test-writer agent", a "security-reviewer agent", or a "documentation agent" — each focused on its job.
Analogy: Instead of hiring one person to build your house, you hire specialists — an electrician, a plumber, a carpenter. Each is an expert in their area. Claude Agents work the same way: parallel specialists that coordinate to complete complex tasks faster.
# .claude/agents/security-reviewer.md
---
name: security-reviewer
description: Reviews code for security vulnerabilities
tools: Read, Grep, WebSearch
---

You are a security expert. Review all code for:
- SQL injection vulnerabilities
- Exposed secrets or API keys
- Insecure authentication patterns
- OWASP Top 10 issues
Report issues with severity levels: Critical / High / Medium / Low
🧪
Test-writer agent Only writes tests. Knows your testing framework deeply. Runs in parallel with the main task.
📚
Documentation agent Reads code changes and automatically keeps docs in sync
🔎
Research agent Searches the web and reads docs so the main Claude doesn't have to

06 / The Plug Standard

What is MCP?

Model Context Protocol — an open standard that lets AI connect to external tools and data sources in a consistent, plug-and-play way.

The best analogy: USB. Before USB, every device used a different cable — printers, cameras, keyboards all had unique connectors. USB created one universal standard so any device works with any computer. MCP does the same thing for AI: instead of each AI tool needing a custom integration with every service, MCP is the universal connector that makes any tool work with any AI.

MCP Servers

Each tool (GitHub, Slack, a database) runs a small "MCP server" — a connector that speaks the MCP language. Once it exists, any MCP-compatible AI can use it immediately.

🤖

MCP Clients

AI tools like Claude Code, Cursor, and others are "MCP clients." They connect to MCP servers and can then use those tools as naturally as any built-in feature.

🌍

Open Standard

MCP is open source and not owned by any single company. Anthropic created it, but anyone can build MCP servers. There's already a huge ecosystem of community-built connectors.

What can Claude actually do with MCP?

🐙
GitHub MCP — "Create a pull request for this bug fix" Claude reads your code, writes the fix, creates a branch, commits, and opens a PR — all through MCP without leaving the chat
🗄️
Database MCP — "How many users signed up last week?" Claude connects directly to your Postgres or SQLite database, writes and runs the SQL query, and shows you the answer
💬
Slack MCP — "Summarize what the team discussed today" Claude reads your Slack channels and gives you a human-readable summary of all the day's conversations
🗺️
Maps MCP — "Find coffee shops near our office" Claude calls the Google Maps API through MCP and returns real, current location results

Adding an MCP server to Claude Code

It's usually one command. For example, to add the GitHub MCP server:

# Add the GitHub MCP server to Claude Code
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Now you can ask Claude things like:
"List my open pull requests"
"Create an issue for the bug I just described"
"Show me the last 5 commits on main"
